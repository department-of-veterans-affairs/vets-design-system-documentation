Race and ethnicity

Use: DeployedWeb
GitHub Issues...FigmaResearch...

Use this pattern when collecting race and ethnicity information from Veterans. Only collect data VA doesn't already possess and ensure collection directly benefits the user experience.

Usage

When to use this pattern

  • Data VA doesn't have. Only collect race and ethnicity data VA doesn't already possess.
  • User benefit. Ensure collection directly benefits the user experience, not just VA operations.

Design principles

  • Ethnicity first. Ethnicity precedes race as a separate, single-response question.
  • Multiple race selections. Race uses checkboxes to allow multiple selections.
  • Include prefer not to answer. Provide 'Prefer not to answer' option for both questions.
  • Mark as optional. Both questions should be clearly labeled as optional.
  • Explain the benefit. Clearly explain how providing this information helps the user.
  • Stay in flow. Do not remove the user from the task at hand to communicate details about this data collection.

When not to use this pattern

  • Contact Design System team first. This pattern faces potential deprecation. Consult the Design System team before implementing.

Examples

Ethnicity question

Single-response question about ethnicity with prefer not to answer option.

Race question

Checkbox question allowing multiple race selections with prefer not to answer option.

How to design and build

The race and ethnicity pattern uses separate questions for each demographic.

How this pattern works

  1. Ethnicity question first: Use radio buttons or single-select for ethnicity.
  2. Race question second: Use checkboxes to allow multiple selections.
  3. Include 'Prefer not to answer' option for both questions.
  4. Mark both questions as optional with clear labeling.
  5. If 'Prefer not to answer' is selected, it negates other selections.
  6. Keep explanatory content brief and policy-compliant within the workflow.

Components used in this pattern

  • Checkbox
  • Radio button

Accessibility considerations

  • Ensure all options are keyboard accessible.
  • Fieldset and legend must properly group related options.
  • Optional field indicators should be announced by screen readers.
