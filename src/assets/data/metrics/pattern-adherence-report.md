## Notes

- **"Ask users for" patterns** have higher adoption because most map to shared platform-level code (`web-component-patterns`) that simple forms teams actively use.
- **"Keep a record of submitted information"** (ConfirmationView) is now the highest-adoption pattern at 33%. An additional 26 forms use confirmation pages but have not yet migrated to the standardized `ConfirmationView` component.
- **"Know when their information is prefilled"** is the newest pattern with the most integration complexity, currently adopted by only 2 forms.
- Compliance percentages are calculated against all 55 tracked forms, including forms that may not need a given pattern (e.g., forms with no phone fields would not be expected to use the phone pattern).
