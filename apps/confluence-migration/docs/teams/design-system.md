# Design System Team

The Design System Team is a platform team responsible for maintaining the VA Design System, component library, and supporting design consistency across all VA digital products.

## Team Mission

**Create and maintain a unified design system that enables VA teams to build accessible, consistent, and Veteran-centered digital experiences efficiently.**

## How to Work with Us

### üéØ **Getting Support**
1. **Check Documentation First**: [Design System Docs](https://design.va.gov/)
2. **Search Existing Issues**: [GitHub Issues](https://github.com/department-of-veterans-affairs/component-library/issues)
3. **Ask in Slack**: #platform-design-system for quick questions
4. **Schedule Office Hours**: For complex discussions or training needs

### üìù **Requesting New Components**
1. **Review Existing Components**: Ensure your need isn't already addressed
2. **Create RFC (Request for Comments)**: Use our [component proposal template](https://github.com/department-of-veterans-affairs/component-library/issues/new?template=component-request.md)
3. **Include User Research**: Usage patterns, user needs, accessibility requirements
4. **Attend Design Review**: Present proposal to team for feedback

### üêõ **Reporting Issues**
- **Component Bugs**: [GitHub Issues](https://github.com/department-of-veterans-affairs/component-library/issues/new)
- **Documentation Issues**: [Documentation Repo](https://github.com/department-of-veterans-affairs/vets-design-system-documentation/issues/new)
- **Urgent Issues**: Slack #platform-design-system with @channel

### ‚ú® **Contributing**
We welcome contributions from the VA community:

1. **Documentation Improvements**: Fix typos, add examples, clarify instructions
2. **Component Enhancements**: Bug fixes, accessibility improvements, new variants
3. **New Components**: Following our contribution guidelines and approval process
4. **Testing**: Help us test components across different browsers and devices
