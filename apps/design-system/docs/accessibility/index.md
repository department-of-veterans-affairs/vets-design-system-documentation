# Accessibility

All VA.gov digital experiences must be accessible to all users, including those with disabilities. This section provides guidelines, requirements, and testing procedures to ensure compliance with accessibility standards.

## Why Accessibility Matters

Veterans and their families include people with a wide range of abilities. Accessibility ensures that:

- Users with visual impairments can navigate with screen readers
- Users with motor impairments can use keyboard navigation
- Users with cognitive disabilities can understand content
- Users with hearing impairments can access audio content

## Standards

VA.gov follows **WCAG 2.1 Level AA** standards, which is required by:

- Section 508 of the Rehabilitation Act
- 21st Century IDEA (Integrated Digital Experience Act)

## Quick Reference

### WCAG 2.1 AA Requirements

| Category | Requirements |
|----------|-------------|
| **Perceivable** | Text alternatives, captions, color contrast, resize text |
| **Operable** | Keyboard accessible, enough time, no seizure triggers, navigable |
| **Understandable** | Readable, predictable, input assistance |
| **Robust** | Compatible with assistive technologies |

### Common Requirements

- **Color contrast**: 4.5:1 for normal text, 3:1 for large text
- **Focus visible**: All interactive elements must have visible focus
- **Labels**: All form inputs must have associated labels
- **Alt text**: All meaningful images must have alt text
- **Headings**: Use proper heading hierarchy (H1 → H2 → H3)
- **Links**: Link text must be descriptive and unique
