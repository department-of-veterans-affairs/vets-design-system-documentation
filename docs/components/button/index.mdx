usage:
  when_to_use:
    - value: '**Actions**: Use buttons for clickable actions such as "Add", "Close",
        "Cancel", or "Save"'
    - value: "**Triggers**: Use buttons to trigger functionality via JavaScript, like
        closing a modal"
    - value: "**Form progression**: Primary buttons advance users to the next step in
        a process"
    - value: "**Secondary actions**: Use secondary buttons for non-primary actions
        that need to be downplayed"
  when_to_consider_else:
    - value: "**Navigation**: For links between pages, use
        [Link](/docs/components/link/) components instead"
    - value: "**Call-to-action prominence**: Consider using [Action
        links](/docs/components/link/action) for visually prominent CTAs linking
        to other pages"
    - value: "**General decoration**: Avoid using buttons purely for visual effect"
  behavior:
    - value: Limit primary buttons per page or section to maintain their impact
    - value: Arrow icons are reserved exclusively for "Back" and "Continue" buttons in
        forms
    - value: Use icons in buttons only when necessary for clarityâ€”never for decoration
    - value: Avoid disabling buttons as a design solution
    - value: Primary buttons typically appear first or to the left of secondary
        buttons

code_usage_mobile: |-
  ```swift
  // Primary button
  Button("Continue", action: handleContinue)
    .buttonStyle(PrimaryButtonStyle())

  // Secondary button
  Button("Cancel", action: handleCancel)
    .buttonStyle(SecondaryButtonStyle())

  // Destructive button
  Button("Delete", role: .destructive, action: handleDelete)
    .buttonStyle(DestructiveButtonStyle())
  ```

  ```kotlin
  // Primary button (Android)
  VAButton(
    text = "Continue",
    onClick = { handleContinue() },
    style = VAButtonStyle.Primary
  )

  // Secondary button
  VAButton(
    text = "Cancel",
    onClick = { handleCancel() },
    style = VAButtonStyle.Secondary
  )
  ```
